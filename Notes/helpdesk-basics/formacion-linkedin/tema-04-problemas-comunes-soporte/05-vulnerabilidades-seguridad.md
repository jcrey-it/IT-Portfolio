# Vulnerabilidades de seguridad

Las vulnerabildades de seguridad representan uno de los mayores riesgos para los usuarios y para la red de una organización. En muchos casos, el punto de entrada no es un fallo técnico, si no el propio usuario.


## 1. Phishing por correo electrónico
La forma más común en la que un atacante puede comprometer una red corporativa es mediante correos electrónicos de phishing.

Situación habitual:
- El usuario recibe un correo que aparenta ser de una fuente confiable
- Hace clic en un enlace o descarga un archivo adjunto
- El equipo comienza a comportarse de forma extraña

Consecuencias posibles:
- Infección por malware
- Compromiso de credenciales si el usuario introduce sus contraseñas en un sitio falso
- Riesgo para el resto de la red corporativa


## 2. Enlaces web maliciosos
Los enlaces web también pueden ser una fuente importante de infección.

Situaciones habituales:
- Anuncios infectados en páginas web
- Enlaces maliciosos dentro de artículos aparentemente legítimos
- Redirecciones automáticas a sitios web maliciosos

Estas acciones pueden provocar la descarga de malware sn que el usuario sea consciente.


## 3. Spyware y sitios web inseguros
Algunas páginas web, como ciertos sitios de juegos, son conocidas por distribuir spyware.

Características del spyware:
- Espía los hábitos de navegación del usuario
- Muestra publicidad dirigida
- Puede ser difícil de eliminar
- No siempre es tan peligroso como otros tipos de malware, pero sigue siendo una amenaza

Aunque su impacto suele ser menor, afecta al rendimiento y a la privacidad del usuario.


## 4. Dispositivos USB infectados
Las unidades USB y otros dispositivos externos también pueden introducir malware en un sistema.

Riesgos asociados:
- Dispositivos USB de origen desconocido
- Malware inyectado durante el proceso de fabricación
- Uso de dispositivos sin escanear previamente

Buenas prácticas:
- Escanear siempre los dispositivos USB antes de usarlos
- Restringir el uso de dispositivos externos no autorizados


## 5. Ingeniería social
La ingeniería social es una técnica en la que un atacante se hace pasar por un empleado o una persona de confianza para obtener información sensible.

Situaciones habituales:
- Llamadas telefónicas falsas
- Solicitudes de contraseñas o información interna
- Suplantación de identidad de empleados o personal de IT

Estas personas pueden ser difíciles de detectar debido a su habilidad para manipular y generar confianza.

La mejor defensa es:
- Verficación de identidad
- Formación y concienciación del personal
- Procedimientos claros para el manejo de información sensible

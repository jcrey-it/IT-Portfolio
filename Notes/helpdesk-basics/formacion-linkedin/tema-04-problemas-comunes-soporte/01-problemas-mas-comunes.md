# Problemas más comunes en soporte técnico

En entornos de soporte técnico, existen una serie de problemas recurrentes que aparecen con frequencía. Conocerlos y saber cómo abordarlos permite resolver incidencias de forma más rápida y eficiente.
Este módulo describe algunos de los problemas más comunes y los pasos habituales para su diagnóstico y resolución.


## 1. Usuario no puede iniciar sesión
Uno de los problemas más frecuentes es que el usuario no pueda iniciar sesión en su equipo o en una aplicación.

**Pasos a seguir:**
- Verificar que la tecla Bloq Mayús no esté activada
- Comprobar si la contraseña ha caducado
- Revisar si la cuenta está bloqueada o suspendida en Active Directory
- Restablecer la contraseña si es necesario

Este proceso permite descartar errores simples antes de aplicar soluciones más avanzadas.


## 2. Restauración de archivos eliminados
Los usuarios pueden reportar la desaparición de archivos importantes.

**Pasos a seguir:**
- Verificar si los archivos se encuentran en la papelera de reciclaje
- Restaurar los archivos desde una copia de seguridad si no están en la papelera
- Utilizar restauración de instantáneas de volumen si está habilitada

Esto permite recuperar información sin afectar al sistema.


## 3. Equipo lento
Otro problema habitual es que el usuario perciba que su equipo funciona lentamente.

**Pasos a seguir:**
- Comprobar el uso de CPU y memoria
- Eliminar archivos temporales del sistema
- Verificar que el disco duro no esté lleno
- Analizar el equipo en busca de virus o malware

Estas comprobaciones ayudan a identificar problemas de rendimiento.


## 4. Conexión a internet lenta
Cuando un usuario reporta lentitud en la conexión, es importante determinar el alcance del problema.

**Pasos a seguir:**
- Verificar sii el problema afecta a toda la organización o solo a un usuario
- Si afecta a todos, revisar proveedor de internet, firewall o infraestructura de red
- Si afecta a un solo usuario, comprobar la configuración de IP y DNS
- Revisar si el sistema utiliza IP estática o DHCP y si hay direcciones disponibles

Esto permite distinguir entre problemas individuales y problemas globales.


## 5. Problemas de impresión
Los problemas con impresoras son muy comunes en soporte técnico.

**Pasos a seguir:**
- Verificar que la impresora esté encendida
- Comprobar que haya papel y tinta
- Revisar si existe algún atasco de papel
- Observar luces o mensajes de error en la impresora
- Instalar o actualizar los drivers de la impresora

Estas acciones cubren la mayoría de incidencias de impresión.


## 6. Acceso a unidades compartidas
En muchas empresas existen unidades de red compartidas.

**Pasos a seguir:**
- Verificar que se puede acceder al equipo del usuario
- Si no es posible, comprobar conectividad y configuración IP
- Si se puede acceder, hacer ping al servidor de archivos
- Comprobar si el servidor está operativo
- Escalar el problema al equipo de servidores si el fallo es general

Esto ayuda a determinar si el problema es local o del servidor.


## 7. Malware en el equipo del usuario
El malware supone un riesgo grave para la organización.

**Pasos a seguir:**
- Desconectar inmediatamente al equipo de la red
- Solicitar al usuario que apague el equipo si está en remoto.
- Retirar el disco duro y llevarlo a un entorno aislado (sandbox)
- Analizar el disco y eliminar el malware o formatearlo si es necesario

Estas medidas ayudan a evitar la propagación del malware.


## 8. Teclado o ratón inalámbrico no funcionan
Los periféricos inalámbricos pueden presentar problemas de conexión.

**Pasos a seguir:**
- Comprobar el estado de las baterías o pilas
- Revisar posibles interferencias con otros dispositivos inalámbricos
- Separar a los usuarios o sustituir el dispositivo por uno con cable

Esto permite resolver rápidamente problemas de hardware básico.


## 9. Pantallazos azules
Los errores de pantalla azul requieren atención presencial.

**Pasos a seguir:**
- Intentar reiniciar el equipo
- Enviar un técnico al lugar s el problema persiste
- Analizar posibles fallos de sistema, hardware o drivers
- Revisar los archivos de volcado generados por el sistema

Esto facilita el diagnóstico del error.


## 10. El equipo no arranca
En algunos casos, el equipo del usuario no inicia correctamente.

**Pasos a seguir:**
- Comprobar que el equipo esté conectado a la corriente
- Verificar que la fuente de alimentación esté encendida
- Presionar botón de reinicio
- Utilizar herramientas de diagnóstico para investigar el fallo

Este enfoque permite descartar problemas básicos antes de profundizar.

# Cómo hablar con usuarios no técnicos

La comunicación con usuarios no técnicos requiere un enfoque distinto al uso habitual del lenguaje técnico. El objetivo principal es que el usuario se sienta cómodo, comprendido y seguro durante el proceso de soporte.


## 1. Crear un entorno de confianza
Cuando se habla de temas técnicos, es importante que el usuario se sienta tranquilo y con la confianza suficiente para expresar dudas.

Buenas prácticas:
- Informar al usuario de que puede pedir aclaraciones si la explicación resulta demasiado técnica
- Mostrar una actitud abierta y accesible
- Explicar las cosas con humildad y sin superioridad técnica

Esto ayuda a reducir la ansiedad del usuario ante problemas tecnológicos.


## 2. Escucha activa y empatía
Prestar atención a lo que el usuario dice es clave para comprender el problema real.

Buenas prácticas:
- Escuchar sin interrumpir
- Responder directamente a las preocupaciones del usuario
- Mostrar empatía ante la frustración que pueda estar experimentando

Comprender la situación del usuario facilita una mejor resolución del problema.


## 3. Uso de ejemplos y experiencias previas
Compartir experiencias similares puede ayudar a que el usuario entienda el proces de resolución.

Buenas prácticas:
- Mencionar casos anteriores con problemas similares
- Explicar por qué se realizan determinadas acciones
- Generar confianza cuando se accede remotamente al equipo del usuario

Esto hace que el usuario se sienta más cómodo durante la intervención.


## 4. Adaptar el formato de la explicación
No todas las personas aprenden de la misma manera.

Buenas prácticas:
- Combinar explicaciones verbales con contenido escrito
- Utilizar elementos visuales siempre que sea posible
- Simplificar la información para facilitar su comprensión

El contenido visual suele ser más fácil de entender y recordar que las explicaciones exclusivamente verbales.


## 5. Uso de lenguaje sencillo
Evitar la jerga técnica mejora la comunicación con usuarios no técnicos.

Buenas prácticas:
- Traducir términos técnicos a conceptos simples
- Asegurarse de que el usuario entiende el contexto de lo que se está haciendo
- Centrarse en el beneficio de la solución y no en los detalles técnicos complejos

Lo que puede ser interesante desde un punto de vista técnico puede no ser relevante para el usuario final.


## Conclusión
Hablar con usuarios no técnicos requiere de paciencia, empatía y claridad. Adaptar el lenguaje, usar ejemplos comprensibles y centrarse en los beneficios de la solución mejora la experiéncia del usuario y refuerza la confianza en el soporte técnico.
